警示牌上写着 warning

2024-05-12 2 min read # 代码质量

先讲一个段子：

程序员、产品经理、美术等一行人一起爬山，爬到山顶后，只有程序员掉落悬崖，死翘翘了。
为什么呢？

因为，警示牌上写着 warning，但程序员平时习惯了无视 warning，最终自由落体死翘翘了 😃

盘点一下，因无视 IDE 警告，引起的Bug

  1. 案例一：

Undefined constant 'fasle'

PHP 历史项目，方法的返回值，return fasle。typo false 单词拼写错误。
其实 IDE 给出了警告「Undefined constant 'fasle'」，本来是可以避免线上事故的。

  1. 案例二：

Methon invocation 'xxx' may produce 'NullPointerException'

  1. 案例三：

Condition 'Const.SYSTEM_USER.equals(id)' is always false

equals 左边常量类型为 String，右边 id 是 Long 类型。

  1. 案例四：

Unnecessay unboxing 'Const.XXX.longValue()'
引起 Sonar 报 Bug

  1. 案例五：

Condition 'XXXEnum.PASS.equals(obj.getStatus())' is always false

equals 左边常量类型为 枚举，右边 status 是 Integer 类型。

  1. 案例六：

Illegal format string specifiel:unknown conversion in %l

记录日志时 format 使用错误。

以上案例，本来都是可以避免的，小则浪费研发的自测时间，测试同学时间，Code Reviewer 时间，大则引志线上事故。所以，「尽量」消灭项目中的警告吧。

写完代码，多看看 IDE 编辑区域右上角，有没有 warning、Error（包括静态代码分析工具），能消息的尽量都消灭，macOS，IDEA 的话，多按按 F2，多使用静态代码分析工具。
